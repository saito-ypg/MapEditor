12番　斎藤友輝(220188)

新機能追加
高さを数値で指定して、直接その高さにできるモードの追加。
それに付随して、数値をスピンコントロールでも設定可能にし、高さとブロックの種類を同時に指定するモードも追加した。
ついでに高さ関係のコントロールは、ラジオボタンで選択してないと触れなくなる（非アクティブになる）ようにした。

〇説明
  操作方法は授業通り+ QもしくはEでカメラ位置上下  
〇工夫点
+ ComboBoxに文字入力できないように、種類をプルダウンリストにした
+ ComboBoxはそれ自体を触ってるか種類変更のモードの時以外は選択不可能にした
+ 遠藤先生のに書いてる内容だと思ってた動きではなかったので、if(HIWORD(wp)==CBN_SELCHANGE)を使ってComboBoxで選んだ瞬間に値が変わるようにした
+ ダイアログを触ってるときはクリックしてもステージに影響を及ぼさないようになった（MainのInput::Updateの前にif(hWnd==GetForegroundWindow())を入れてアップデート止めるくらいしかわからなかった。要改善?）
　→ダイアログプロシージャにWM_ACTIVATE入れて、ダイアログがアクティブウィンドウになってたらstage::Updateは抜けるように改善。
(Undo/RedoはList作ってイテレータで～みたいな感じにできそうと思ったが、完成させられなかったためUndoブランチに残した)
〇感想
マウスクリックした位置をワールド座標へ変換するとき、スクリーン座標のx,yはそのまま使ってzだけずらした2つの座標を用意する、というのが自分では思いつかなかったと思う。
製作しようと考えていたゲームの要素に「クリックした位置に向かって移動」というのがあったので、今回の内容を活かして実装してみようと思った。
